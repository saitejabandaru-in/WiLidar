RF Channel State Information (CSI)

Channel State Information captures the subcarrier amplitude and phase dynamics of the WiFi transmission link, describing how signals scatter, reflect, and fade. For each subcarrier index \(f\) at time index \(t\), the raw channel response \(H(f, t)\) is formulated as:

\[H(f, t) = |H(f, t)| \cdot e^{j \angle H(f, t)}\]

Here, \(|H(f, t)|\) represents the amplitude attenuation and \(\angle H(f, t)\) denotes the phase variance. When a subject walks through the Fresnel zone, human body reflections induce phase perturbations. By calculating the variance against the baseline calibration vector \(\bar{H}(f)\), we isolate spatial motion:

\[\Delta H(f, t) = H(f, t) - \bar{H}(f)\]

These multi-path signals are filtered via Hampel outlier rejection and bandpass filters. We apply Principal Component Analysis (PCA) to project the 52-dimensional subcarrier vectors onto the top 3 eigenvalues to reduce signal correlation noise:

\[\mathbf{Y}_{t} = \mathbf{W}^T \mathbf{X}_{t}\]

where \(\mathbf{X}_{t}\) represents the filtered subcarrier vector at time \(t\), and \(\mathbf{W}\) is the projection matrix composed of the top three dominant eigenvectors extracted from the channel covariance matrix.

WiFi Sensing Pipeline Architecture

Hover over each module to explore how physical movements translate to digital coordinates

1. Raw CSI Acquisition

ESP32-S3 firmware captures subcarrier amplitude and phase packets at 100Hz frequency.

Channel State Information
A vector representing the signal attenuation and phase shift across 52 individual subcarriers in the WiFi channel.

2. Subcarrier Filtering

Hampel filters discard impulse subcarrier noise. Bandpass filters isolate human breath/step frequencies.

Frequency Filtering
Removes high-frequency electronic noise and static reflections. Limits signals to the 0.1-10 Hz bandwidth representing human motion.

3. PCA Dimensionality

Principal Component Analysis extracts the top 3 dominant eigenvalues, reducing noise.

Eigenvalue Reduction
Extracts orthogonal signal variances to focus on physical movement vectors and compress the vector dimensions from 52 down to 3.

4. ML Position Inference

A calibrated Random Forest model maps PCA trajectories to exact metric X, Y room coordinates.

Regression Coordinates
Supervised models match the PCA amplitude patterns to physical rooms and coordinates validated by lidar base trackers.

Integration & Deployment Guide

Set up and flash your own physical WiLidar tracking nodes

Flashing Node Firmware

Nodes run custom firmware based on the Espressif ESP-IDF framework to log Channel State Information (CSI) packets directly from physical beacons.

1. Connect your ESP32-S3 module via USB-C, then flash using the Python CLI tool:

pip install esptool esptool.py --chip esp32s3 write_flash 0x10000 hardware/esp32s3/firmware.bin

2. Once flashed, the node broadcasts an AP configuration server. Open your phone, connect to WiLidar-Node-XXXX, and connect it to your local 2.4GHz router.

Transmitter / Receiver Node Setup

Proper node layout is critical to avoid radio blind spots and ensure high spatial coverage.

Primary Configuration: Place one Transmitter node (TX) and one Receiver node (RX) on opposite corners of the room, approximately 1.2 to 1.5 meters above the ground (aligned with human torso heights).

Ensure there are no large metal barriers (like refrigerators) directly blocking the direct line-of-sight path between the two nodes.

Sensing Model Calibration

To train the coordinate models, map the baseline signal profile of the room when it is empty, then walk along marked physical coordinates.

1. Ensure room is empty, click the Save baseline calibration marker in the CLI tool.

2. Walk standard perimeter paths in the room. The system will correlate subcarrier phase variances with your coordinate mapping, saving the results in a local database.

3. Click Retrain Models in the coordinate deck to train the Random Forest regressions.

FastAPI Developer Integration

WiLidar exposes a high-performance REST API to fetch real-time spatial coordinates, update configuration parameters, and retrain the underlying tracking models.

GET /api/status

Retrieve the active spatial positioning data, tracking confidence, and node connectivity states.

{ "status": "active", "active_nodes": 2, "inference_latency_ms": 8.4, "coordinates": {"x": 2.45, "y": 3.12, "z": 0.0}, "confidence": 0.94 }
POST /api/configure

Update subcarrier acquisition settings, outlier filtering thresholds, and node calibration geometries.

{ "hampel_threshold": 3.0, "pca_components": 3, "smoothing_window": 10 }
POST /api/model/retrain

Trigger an asynchronous training cycle on the mapped baseline and coordinate data.

{ "dataset_id": "session_2026_06_04", "estimators": 100, "max_depth": 15 }
